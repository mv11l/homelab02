# Homelab02

Use scaleway cpu instances to run Talos multi-cluster.

## ROADMAP

- Init terraform (terragrunt) project on specific region
- Add talos image disk generated by <https://factory.talos.dev> to object bucket storage.
- Create Vms instances with VPC/Network policies/Public ip/LB using image disk stored on bucket.
- Add Cilium
- Add scaleway sci storage class to cluster.
- ...

## Usage

Requires `tofu` or `terraform` in PATH. Targets auto-detect which binary to use.

```bash
# Initialize providers and backend
make init

# Preview changes
make plan

# Apply infrastructure and run Talos inspection
make apply

# Run Talos inspection against the provisioned instance (requires talosctl)
make talos-inspect
```

`make apply` automatically runs `talos-inspect` after a successful apply, which fetches the public IP from the tofu state and runs:

```bash
talosctl -n <public_ip> version --insecure
talosctl -n <public_ip> get disks --insecure
talosctl -n <public_ip> get links --insecure
```

If `talosctl` is not installed, the install guide is printed:
<https://docs.siderolabs.com/talos/v1.13/getting-started/talosctl>
